翻译有些问题，其实指的是用 N 个 double 构造一个 BST 和 BinarySearchST 的速度对比。
Get 速度 BST 是不会比 BinarySearchST 快的。（1.39lg⁡N1.39\lg N1.39lgN>lg⁡N\lg NlgN）

二叉搜索树一次查找平均需要 1.39lg⁡N1.39\lg N1.39lgN 次比较，二分查找则是 N/2N/2N/2，于是可以求得开销：
二叉查找树：1.39∑i=1N−1lg⁡i=1.39lg⁡(N−1)!=1.39(N−1)lg⁡(N−1)1.39 \sum_{i=1}^{N-1} \lg i=1.39 \lg (N-1)!=1.39(N-1)\lg(N-1)1.39∑i=1N−1​lgi=1.39lg(N−1)!=1.39(N−1)lg(N−1)。
二分查找实现的符号表：1/2+2/2+⋯+(N−1)/2=N(N−1)/41/2+2/2+ \cdots+(N-1)/2=N(N-1)/41/2+2/2+⋯+(N−1)/2=N(N−1)/4 。
令两式相等，可以求得快 10 倍，100 倍，1000 倍的 NNN 值。
例如快 10 倍的方程：

13.9(N−1)lg⁡(N−1)=N(N−1)/413.9lg⁡(N−1)=N/413.9(N-1)\lg(N-1)=N(N-1)/4 \\ 13.9\lg (N-1)=N/4 13.9(N−1)lg(N−1)=N(N−1)/413.9lg(N−1)=N/4

这是一个超越方程，可以简单用程序穷举出一个数值解。

for (var i = 0d; i < int.MaxValue; i++)
{
    if (13.9 * Math.Log2(i - 1) < i / 4)
    {
        Console.WriteLine(i);
        return;
    }
}

解得的三个 N 值分别为 499，7115，91651。

除了基本的新元素赋值外，二叉树在插入时只需要进行比较即可。
但二分查找实现的符号表还需要维持数组有序，需要额外的赋值操作。
